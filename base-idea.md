Eсть идея по разработке и запуске локального приложения по типу api кластера, который будет работать с более крупными моделями, выгруженными с https://huggingface.co/models
Данный кластер имеет структуру 1 ко многим, что означает 1 основной узел и множество вспомогательных которые будут подключены в локальной или интернет сети для распараллеливания больших моделей.

Основной узел должен иметь возможности (python3 + pytorch):
1. подключать вспомогательные узлы по ip
2. получать данные о доступных на них ресурсах
3. выгружать модели с https://huggingface.co с подключением трансформера
4. разделение модели между ресурсами узлов для холодного старта
5. распараллеливание задачи поданную в разделённую модель
Вспомогательный узел должен иметь функционал (python3 + pytorch):
1. Поднятия API сервиса для коммуникации с основным узлом
2. Получение данных для холодного старта (загрузка матрицы являющейся часть модели в RAM или VRAM)
3. Получение данных для обработке на данных холодного старта
4. Отвечать по готовности

Коммуникацию осуществлять по сокетам или более быстрым его аналогам
Предусмотреть возможность становления основного узла в качестве кастомного API провайдера для openclaw

Основной язык пректа: Python3
Основной фреймворк для подключения и использования моделей PyTorch
